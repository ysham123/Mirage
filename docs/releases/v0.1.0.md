# Mirage v0.1.0

`v0.1.0` is the first public alpha release of Mirage.

This release establishes Mirage as a practical path for testing and gating
agent side effects in local development, `pytest`, and CI.

## Highlights

- `MirageSession` is now the canonical Python integration path for agent
  workflows.
- Run-level CLI gates are in place with `summarize-run`, `gate-run`, and
  `validate-config`.
- The bundled procurement harness gives a realistic onboarding flow instead of
  isolated request demos.
- The trace-backed review console is available both through the original
  API-backed surface and the Next.js client in `ui/`.
- The repository now has the baseline open-source scaffolding expected for a
  public project: MIT license, contributing guide, security policy, changelog,
  issue templates, PR template, and docs hub.

## Best Fit Today

Mirage is strongest today when you want to:

- intercept outbound agent HTTP calls in tests or CI
- apply config-driven policy checks before real side effects happen
- keep run traces deterministic and easy to review
- gate risky runs in `pytest` or GitHub Actions

## Current Boundaries

- Mirage is still alpha software. The Python integration and CLI gate path are
  the most mature surfaces.
- The review UI is useful for debugging and demos, but it should still be
  treated as an alpha console, not a finished product shell.
