# Mirage v0.1.2

`v0.1.2` is a first-run UX release. One behavior change, several
documentation improvements.

The motivating bug: a developer who ran their `MirageSession` code without
first starting the Mirage proxy got a 40-line `httpx.ConnectError` traceback
that did not point at the real cause. Mirage now catches that specific
failure and surfaces an actionable message.

## Highlights

- **New error: `MirageProxyUnreachableError`.** `MirageSession.request` now
  detects a connection failure against the configured proxy `base_url` and
  raises `MirageProxyUnreachableError` with a remediation hint:

  ```
  Mirage proxy not reachable at http://localhost:8000. Start it in a separate
  terminal with: python -m uvicorn mirage.proxy:app --host 127.0.0.1 --port 8000
  ```

  Connection failures to any other host:port pass through unchanged, so
  legitimate external errors are not masked. Exported from the public
  `mirage` package as `MirageProxyUnreachableError`.
- **Rewritten "See It In 60 Seconds" README block** as a pip-install-only
  demo against the bundled default mocks and policies. No repo clone
  required.
- **`docs/FIRST_INTEGRATION.md` and `docs/CI_INTEGRATION.md`** now use
  `pip install mirage-ci` instead of the source install. The GitHub Actions
  recipe previously failed when copied into an unrelated user repo because
  there was no `Makefile` in the user's tree.
- **`examples/procurement_harness/README.md`** is honest about requiring a
  repo checkout and points integration readers at
  `docs/FIRST_INTEGRATION.md`.

## Upgrade Notes

No breaking changes. Existing `MirageSession` integrations, CLI commands
(`summarize-run`, `gate-run`, `validate-config`), policy files, and trace
artifacts are unaffected.

Users who were catching `httpx.ConnectError` from `MirageSession` calls
against a dead proxy should consider catching `MirageProxyUnreachableError`
(or its `ConnectionError` base) instead, since the raised type has changed
for that specific scenario. Connection failures to any other host still
surface as `httpx.ConnectError`.

Upgrade with:

```bash
pip install --upgrade mirage-ci
```

## Current Boundaries

- Mirage is still alpha software. The Python integration and CLI gate
  path are the most mature surfaces.
- The review UI remains an alpha console surface, not a finished product
  shell.
