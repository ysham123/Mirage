# Mirage v0.1.1

`v0.1.1` is a documentation and packaging release. No behavior changes.

This release fixes a broken quickstart install path, establishes the
`mirage-ci` name on PyPI, and sharpens the README positioning against
adjacent Python HTTP-testing tools.

## Highlights

- Quickstart now opens with `pip install mirage-ci` and states the
  Python 3.11+ requirement up front, replacing a source-install command
  that failed on clean virtual environments.
- First release published to PyPI under the `mirage-ci` distribution name
  (imports remain as `from mirage import ...`).
- New `Mirage vs. Adjacent Tools` section in the README covers
  `pytest-httpx` / `respx`, `pytest-httpserver`, `VCR.py`, `responses`,
  `WireMock` / `mitmproxy`, and runtime LLM-judge guards, followed by a
  `When not to use Mirage today` bullet list.
- New `Source install` subsection under Contributing documents the
  editable-from-source path for contributors.

## Upgrade Notes

No API or configuration changes. Existing `MirageSession` integrations,
CLI commands, policy files, and trace artifacts are unaffected. Users
on `0.1.0` source installs can upgrade with
`pip install --upgrade mirage-ci` once the package is published.

## Current Boundaries

- Mirage is still alpha software. The Python integration and CLI gate
  path are the most mature surfaces.
- The review UI remains an alpha console surface, not a finished product
  shell.
