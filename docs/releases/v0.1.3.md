# Mirage v0.1.3

`v0.1.3` is a console redesign release. The Mirage review console
(`ui/`) gets a full visual rewrite to a precision-instrument
aesthetic. The Python package itself is unchanged in behavior since
0.1.2 — internal cleanup only.

## Highlights

- **Review console redesign.** The console drops the prior cyan/sora
  typography for a Bricolage Grotesque + Geist + JetBrains Mono
  pairing with a single NVIDIA-toned green accent. Header chrome is
  tighter, the run context strip sticks, the stats bar is denser, and
  the top bar carries an explicit environment indicator. The README
  screenshot has been regenerated against the new console.
- **README badges.** PyPI version and Python version badges added
  above the fold so visitors can see the published distribution at a
  glance instead of digging into `pyproject.toml`.
- **`src/` shim removed.** The repo previously shipped a `src/` tree
  that re-exported every module from `mirage.X` via wildcard. Tests,
  CI, and the Dockerfile all imported from `src` even though the
  PyPI distribution exposed `mirage` as the public path. `src/` is
  now gone; tests, CI, and Docker import from `mirage` directly.

## Upgrade Notes

No breaking changes for `pip install mirage-ci` users. The public
import path was, and remains, the `mirage` package:

```python
from mirage import MirageSession, MirageProxyUnreachableError
```

The `src/` package was never part of the published distribution as a
public API. Internal scripts that imported `from src.mirage.X import
...` (against a repo checkout) need to be updated to import from
`mirage` directly.

Upgrade with:

```bash
pip install --upgrade mirage-ci
```

## Current Boundaries

- Mirage is still alpha software. The Python integration and the CLI
  gate path are the most mature surfaces.
- The review console has been redesigned but is still an alpha surface,
  not a finished product shell.
